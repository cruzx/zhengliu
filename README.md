# 一个名叫郑六的设计师

`zhengliu` 是一个更偏设计师工作流的蒸馏 Skill。

它的核心不是“总结”，而是把一个人、一类评审风格、一套业务判断、或者你自己的工作方法，提炼成后面真的能复用的设计方法论和 Skill。

它尤其适合这些场景：

1. 提炼某个 leader 的过稿标准
2. 提炼某个设计师为什么总能把图做得更容易过
3. 把一类业务问题整理成稳定的设计判断框架
4. 复盘你自己最近反复在做的设计工作
5. 帮你判断一个问题现在该回图、回标注、回文案，还是回一个未来判断

## 它到底是干嘛的

你可以把它理解成一个“设计师工作方法提炼器”。

它能帮你做三类事：

1. 提炼人
- 提炼 leader、评审人、设计师的判断标准
- 看他们为什么觉得某个方案好，为什么觉得某个方案不行
- 总结他们更看重信息清晰、转化效率、氛围表达，还是策略站位

2. 提炼事
- 提炼一类设计任务怎么更容易过稿
- 比如商业化设计、概念版、会员权益、活动页、播放器包装、收银台包装
- 把零散评审意见，变成一套更稳定的判断逻辑
- 根据线上数据回溯，判断当前设计稿
- 
3. 提炼你自己
- 复盘你最近反复在做什么工作
- 看 leader 们对业务的判断标准是什么
- 看你面对一个问题时，最该输出的是图、标注、文案，还是未来判断

## 它适合谁

这个 Skill 最适合：

- 设计师
- 交互设计师
- 视觉设计师
- 商业化设计师
- 需要频繁过稿、评审、汇报的人
- 想把自己的工作方法沉淀下来的人

## 最常见的设计师用法

### 1. 提炼 leader / 评审人的标准

```text
用 zhengliu 提炼我 leader 的过稿标准，重点看他对商业化设计的判断逻辑。
```

适合回答：

- 他最在意什么
- 他觉得什么样的图是“有价值”的
- 他对“好结果”和“差结果”的区分点是什么
- 以后我给他回稿时，该优先展示什么

### 2. 提炼某类设计工作的过稿逻辑

```text
用 zhengliu 提炼“概念版首页怎么更容易过稿”这套逻辑，做成一个可复用 skill。
```

适合提炼：

- 概念版
- 商业化设计
- 会员触点
- 支付挽留
- 权益感知
- 活动页包装
- 播放器创意

### 3. 提炼一个设计师的思维方式

```text
用 zhengliu 蒸馏 XXX，重点看他的设计判断、表达方式和过稿能力。
```

最后适合产出：

- 这个人的方法论
- 这个人的评审风格
- 这个人的表达话术
- 一个可以继续调用的 `SKILL.md`

### 4. 复盘你自己

```text
用 zhengliu 蒸馏我自己，重点看我最近反复做的设计工作、leader 对我的评价、以及我该怎么放大自己的优势。
```

这个模式更像“设计师自我蒸馏”：

- 帮你看你真正高频在做什么
- 帮你看你在哪类问题里最有方法
- 帮你把零散经验整理成稳定能力

### 5. 判断现在该回什么

```text
我给你一个设计问题，你用 zhengliu 帮我判断现在该回图、回标注、回文案，还是回一个未来判断。
```

这类场景很适合现在的 `zhengliu`，因为它已经不只是提炼信息，而是会优先帮你做“输出类型分流”。

## 它会产出什么

根据输入不同，它通常会产出这些东西里的一个或多个：

- 一个新的设计师导向 Skill
- 一份 leader / 评审标准提炼
- 一份改稿优先级清单
- 一份设计评审逻辑
- 一份更适合汇报的表达框架
- 一份“这题现在该回什么”的判断

如果你给的是设计稿、截图、评审记录，它更偏向输出：

- 标注建议
- 改稿建议
- 方案收敛方向
- 回评审的话术

如果你给的是聊天记录、文档、语音转写，它更偏向输出：

- 风格提炼
- 判断标准
- 工作方法
- 可复用的 `SKILL.md`

## Repo Structure

```text
.
├── SKILL.md
├── README.md
├── references/
│   ├── extraction-framework.md
│   ├── skill-template.md
│   └── research/
└── scripts/
    ├── download_subtitles.sh
    ├── merge_research.py
    ├── quality_check.py
    └── srt_to_transcript.py
```

说明：

- `SKILL.md`：Skill 主体，定义触发方式、工作流、设计师优先模式、产出结构与约束
- `references/`：蒸馏框架、模板、研究材料
- `scripts/`：字幕下载、转写清洗、研究汇总、质量检查

## Install

### Codex

把整个仓库目录放进你的 skill 目录，并确保目录名就是 `zhengliu`：

```bash
git clone <your-repo-url> zhengliu
cp -R zhengliu ~/.codex/skills/zhengliu
```

如果你已经在本地 clone 好，也可以直接复制：

```bash
cp -R /path/to/zhengliu ~/.codex/skills/zhengliu
```

### 其他本地 Skill 目录

如果你不是用 Codex，也可以把目录复制到你自己的 skill 根目录中，只要最终结构保持：

```text
<skills-root>/zhengliu/
  ├── SKILL.md
  ├── references/
  └── scripts/
```

## Notes

- `references/research/` 里的模板和研究文件属于 skill 结构的一部分，不建议随意删除。
- `scripts/` 主要用于字幕下载、转写清洗、研究汇总和质量检查。
- 这个仓库本身就是 skill 主体，不需要再套一层 `zhengliu/` 子目录。
- 这个版本的 `zhengliu` 已经更偏向设计师思维逻辑，不只做总结，更强调评审、过稿、收敛和设计判断。
